# HyperFlow Evaluator Session Checklist

## Before the session

- [ ] Confirm the current branch/build is the intended demo baseline
- [ ] Run `pnpm run verify`
- [ ] Open `http://localhost:4173/examples/vanilla-starter/index.html`
- [ ] Clear old notes and prepare the feedback template
- [ ] Confirm the evaluator has not already been coached on the expected story

## At the start

- [ ] Introduce the session as a short product-demo evaluation
- [ ] Keep the session to a consistent target length (for example 10-15 minutes)
- [ ] Do **not** explain the intended answer immediately
- [ ] Ask the evaluator to think aloud while using the demo

## During the first 30 seconds

- [ ] Note the first thing they say the demo is “about”
- [ ] Note whether they understand what to click first
- [ ] Note whether scenario names feel clear or internal
- [ ] Note whether summary cards help or confuse
- [ ] Capture any hesitation, silence, or backtracking

## During the rest of the walkthrough

- [ ] Watch whether the guided steps reduce guesswork
- [ ] Watch whether they understand the value of pan/zoom + hit-test proof
- [ ] Record any places where copy, labels, or hierarchy feel misleading

## After the session

- [ ] Ask the evaluator to summarize what HyperFlow was proving
- [ ] Ask what felt clearest
- [ ] Ask what felt confusing or internal
- [ ] Save the filled feedback template
- [ ] Tag the session outcome as success / mixed / confusion
